# Moment Track — 기획서

> 비디오 기반 멀티 오브젝트 트래킹 및 Re-ID 데이터셋 레이블링 툴

---

## 1. 프로젝트 개요

**Moment Track**은 비디오 시퀀스에서 멀티 오브젝트 트래킹(MOT) 및 Re-Identification(ReID) 데이터셋을 효율적으로 제작하기 위한 오픈소스 어노테이션 툴이다.

로컬 GPU를 활용한 AI 보조 레이블링을 지원하며, 클라우드 의존 없이 완전히 로컬 환경에서 동작한다.

---

## 2. 배경 및 문제 정의

### 2.1 현대 컴퓨터 비전의 데이터 요구

현대 컴퓨터 비전 시스템은 점점 더 비디오 기반 데이터셋에 의존하고 있다.

- 멀티 오브젝트 트래킹 (MOT)
- 사람 / 차량 Re-Identification (ReID)
- 로보틱스 인식
- 행동 분석
- 자율주행 데이터셋

### 2.2 기존 툴의 한계

CVAT, Label Studio, Roboflow 등 기존 어노테이션 툴은 **이미지 레벨 레이블링**에 최적화되어 있다.

| 문제 | 설명 |
|---|---|
| 이미지 중심 설계 | 프레임을 독립적으로 취급하여 시간적 연속성 관리가 어려움 |
| Identity 개념 부재 | 동일 객체의 여러 트랙을 하나의 개체로 묶는 기능이 없음 |
| ReID 워크플로우 미지원 | 크롭 갤러리, identity 검증 등 ReID 특화 기능 없음 |
| 클라우드 의존 | 민감한 데이터(보안, 의료, 산업 영상)를 외부로 보내야 하는 문제 |

### 2.3 핵심 통찰

> 기존 툴은 비디오를 "이미지의 연속"으로 취급한다.  
> Moment Track은 비디오를 "시간과 정체성의 그래프"로 취급한다.

---

## 3. 핵심 개념

Moment Track은 **시간(Time)**과 **정체성(Identity)**을 일급 시민(first-class citizen)으로 다룬다.

### 3.1 데이터 모델

```
Identity (실세계 객체)
 ├─ Track 1 (프레임 0–120, 카메라 A)
 ├─ Track 2 (프레임 300–420, 카메라 A)
 └─ Track 3 (카메라 B)
```

| 개념 | 정의 |
|---|---|
| **Identity** | 실세계의 단일 객체 (사람, 차량, 로봇 등) |
| **Track** | 프레임 시퀀스 위에서 해당 객체의 궤적 |
| **Detection** | 단일 프레임에서의 바운딩 박스 또는 마스크 |

### 3.2 이 구조가 가능하게 하는 것

- Identity 수정 (identity switch 교정)
- 트랙 병합 / 분리
- 크로스 카메라 트래킹 (향후)
- ReID 데이터셋 생성

---

## 4. 핵심 기능

### 4.1 비디오 네이티브 어노테이션

- 비디오 타임라인 탐색
- 프레임 스크러빙
- 트랙 시각화
- 키프레임 어노테이션
- 프레임 간 보간(interpolation)
- 궤적 편집

### 4.2 Track / Identity 분리 모델

- 동일 객체에 속하는 트랙 병합
- Identity switch 분리
- 트래킹 드리프트 교정

### 4.3 ReID Crop Gallery

각 Identity에서 추출된 크롭 이미지를 갤러리 형태로 표시.

- 크롭 비교
- Identity 병합
- Identity switch 감지

### 4.4 로컬 GPU 자동 레이블링

클라우드 없이 로컬 GPU로 AI 보조 어노테이션 수행.

| 단계 | 지원 모델 |
|---|---|
| Detection | YOLO 계열 |
| Tracking | ByteTrack / BoT-SORT |
| ReID Embedding | OSNet / FastReID |
| Segmentation | SAM 계열 (향후) |

워크플로우:
```
모델 선택 → 인퍼런스 실행 → 어노테이션 생성 → 사람 검토
```

### 4.5 데이터셋 내보내기

| 포맷 | 용도 |
|---|---|
| YOLO | Detection 학습 |
| COCO | Detection 학습 |
| MOT Challenge | 트래킹 학습 |
| ReID 이미지 폴더 | Re-ID 학습 |

## 6. 시스템 구성

```
브라우저 (React)
     │
     ▼
로컬 API 서버 (FastAPI)
     │
     ├─── SQLite DB
     │
     └─── GPU 인퍼런스 워커 (별도 프로세스)
               │
               ▼
          로컬 GPU
```

### 기술 스택

| 영역 | 선택 |
|---|---|
| 프론트엔드 | React |
| 백엔드 | Python / FastAPI |
| 데이터베이스 | SQLite |
| AI 워커 | Python (별도 프로세스) |
| 워커 통신 | WebSocket / SSE |
| 실행 방식 | 로컬 웹 서버 (Python으로 실행) |
| Repo | Monorepo |
