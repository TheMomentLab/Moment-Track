### 3.2 유효성 검사

| 조건 | 동작 |
|---|---|
| 프로젝트명 비어 있음 | "프로젝트 생성" 버튼 비활성화 |

### 4.2 VideoCanvas 인터랙션

#### 도구 모드

| 도구 | 단축키 | 동작 |
|---|---|---|
| 선택 (Select) | `V` | 박스 클릭 선택, 드래그 이동, 핸들 크기 조절 |
| 박스 그리기 (Box) | `B` | 캔버스 드래그 → 박스 생성 → 클래스 선택 팝업 |
| 키프레임 지정 | `K` | 선택된 Detection을 키프레임으로 토글 |

#### 박스 선택 상태

| 상태 | 시각적 표현 |
|---|---|
| 비선택 | 반투명 색상 테두리 + 클래스 라벨 |
| 선택됨 | 흰 테두리 + 리사이즈 핸들 8개 표시 |
| 키프레임 | 테두리 두께 증가 + 다이아몬드 아이콘 |
| 보간됨 | 점선 테두리 |

### 4.3 TrackList (사이드 패널)

- 트리 구조: Identity > Track
- 색상: 각 Identity에 고유 색상 할당 (비디오 오버레이와 동일)
- 아이템 클릭: 해당 Track의 시작 프레임으로 이동
- 아이템 우클릭 메뉴:
  - **Identity에서**: 라벨 변경, Identity 삭제
  - **Track에서**: Identity 재할당, Track 분리, Track 삭제

### 4.4 Timeline

```
[0]──────────────[현재 위치▼]──────────────[END]
Identity#1: ████████████████          ████████████
Identity#2:          ██████████████████
키프레임:         ◆          ◆    ◆
```

| 요소 | 설명 |
|---|---|
| 스크러버 | 드래그하여 프레임 이동 |
| Identity 트랙 바 | 해당 Identity가 존재하는 프레임 구간 색상 표시 |
| 키프레임 마커 | 다이아몬드(◆) 아이콘, 클릭 시 해당 프레임으로 이동 |
| 줌 | 마우스 휠로 타임라인 확대/축소 |

## 8. 공통 UI 컴포넌트

### 8.1 TopBar

| 요소 | 위치 | 동작 |
|---|---|---|
| 메뉴 버튼 | 좌측 | 드롭다운: 프로젝트 홈, 설정 |
| 브레드크럼 | 중앙 | 프로젝트명 / 비디오명 |
| 저장 상태 | 중앙-우측 | "저장됨 ✓" / "저장 중..." |
| AI 레이블링 버튼 | 우측 | 인퍼런스 패널 열기 |
| 워커 상태 | 우측 | 녹색●(Idle) / 노란색●(Running) / 빨간색●(Error) |
| Crop Gallery | 우측 | 갤러리 화면 이동 |
| 내보내기 | 우측 | 내보내기 다이얼로그 열기 |

### 8.2 토스트 알림

- 위치: 화면 우하단
- 자동 닫힘: 3초 (오류는 유지)
- 종류: `success` (초록), `error` (빨강), `info` (파랑), `warning` (노랑)

### 8.3 확인 다이얼로그

위험한 작업(삭제, 병합 등) 전 표시되는 모달.

```
┌───────────────────────────────────┐
│ Identity 병합                      │
│                                   │
│ Identity #3의 모든 트랙이           │
│ Identity #1로 이동됩니다.           │
│ 이 작업은 되돌릴 수 없습니다.        │
│                                   │
│              [취소] [병합 확인]    │
└───────────────────────────────────┘
```

### 8.4 컨텍스트 메뉴

우클릭 시 표시되는 팝업 메뉴.

**TrackList 아이템 (Track)**:
- Identity 재할당...
- 현재 프레임에서 분리
- 트랙 삭제

**TrackList 아이템 (Identity)**:
- 라벨 변경
- Identity 삭제 (트랙 유지)

**VideoCanvas 박스**:
- 클래스 변경
- 키프레임 토글
- 이 Detection 삭제
- 이 프레임의 모든 Detection 삭제

## 10. 색상 체계

### 10.1 Identity 색상

각 Identity에는 고유한 색상이 자동 할당된다.

## 11. 기술 스택 — 프론트엔드 라이브러리

### 11.1 개요

Moment Track 프론트엔드는 **React + Zustand + Tailwind CSS** 기반이며,
아래 라이브러리 조합으로 구현한다.

| 역할 | 라이브러리 | 비고 |
|---|---|---|
| UI 컴포넌트 | **shadcn/ui** (Radix + Tailwind) | 접근성 내장, 코드 복사 방식, 완전 커스텀 가능 |
| 비디오 캔버스 오버레이 | **react-konva** | CVAT · Label Studio에서 검증된 바운딩 박스 + Transformer 패턴 |
| 패널 레이아웃 | **react-resizable-panels** | shadcn/ui `Resizable` 컴포넌트로 래핑, VS Code 스타일 패널 |
| 트리 뷰 (Identity > Track) | **react-arborist** | 가상화·드래그앤드롭·키보드 내비게이션 내장 |
| 키보드 단축키 | **react-hotkeys-hook** | React Hook 네이티브, 주간 다운로드 227만 |
| 컨텍스트 메뉴 | **@radix-ui/react-context-menu** | shadcn/ui에 포함, WAI-ARIA 접근성 |
| 타임라인 | **커스텀 구현** (canvas 기반) | 어노테이션 전용 타임라인은 기성 라이브러리 없음 |
| 상태 관리 | **Zustand** | 경량, 보일러플레이트 최소 |
| 스타일링 | **Tailwind CSS** | 유틸리티 퍼스트, 다크 테마 기본 |

### 11.2 선정 근거

#### shadcn/ui

- Radix Primitives 기반 → Dialog, Select, Slider, Toast, Progress 등 접근성 내장 컴포넌트 제공
- 코드 복사 방식 → 불필요한 번들 없음, 완전한 커스터마이징 자유도
- Tailwind 네이티브 → 프로젝트 스타일링 체계와 일관성 유지
- 대안 검토: Ant Design(CVAT 사용, 엔터프라이즈급이나 번들 큼), Mantine(올인원이나 Tailwind 통합 약함), MUI(Material 느낌이 데스크톱 도구와 안 맞음)

#### react-konva

- CVAT, Label Studio 모두 사용 → 바운딩 박스 어노테이션 도구의 사실상 표준
- `Stage > Layer > Rect + Transformer` 구조로 선택·이동·리사이즈 핸들 구현
- React 선언형 패턴과 자연스럽게 통합 (vs Fabric.js의 명령형 API)

#### react-arborist

- 가상화(react-window 기반) → 수천 노드에서도 성능 저하 없음
- Apache Superset이 Ant Design Tree의 성능 이슈로 react-arborist로 교체한 사례
- Docmost, Atuin Desktop 등 프로급 도구에서 사용
- 드래그앤드롭, 인라인 편집, 커스텀 노드 렌더러 지원

#### react-hotkeys-hook

- React Hook 네이티브 → `useHotkeys('mod+z', undo)` 패턴
- 스코프·포커스 핸들링 지원 → 도구 모드별 단축키 전환에 적합
- `key` 기반 → 국제 키보드 레이아웃 호환
- Twenty CRM, Langflow, Teable 등 프로급 앱에서 사용
- 대안 검토: tinykeys(650B로 가볍지만 React 훅 아님), TanStack Hotkeys(초기 단계)

#### 타임라인 (커스텀)

- 요구사항: 스크러버 + 다중 Identity 트랙 바 + 키프레임 마커(◆) + 줌
- GitHub에서 "annotation timeline keyframe" 조합 검색 결과 없음 → 기성 라이브러리 부재
- CVAT도 타임라인은 커스텀 구현
- `<canvas>` 또는 react-konva의 별도 Stage로 직접 구현 예정
