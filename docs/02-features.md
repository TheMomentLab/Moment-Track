## 2. 프로젝트 관리

### 2.1 프로젝트 생성

| 항목 | 내용 |
|---|---|
| **설명** | 새 어노테이션 프로젝트를 생성한다. |
| **입력** | 프로젝트 이름, 비디오 파일 경로 (1개 이상), 클래스 목록 |
| **출력** | 프로젝트 디렉토리 및 SQLite DB 파일 생성 |
| **조건** | 지원 비디오 포맷: `.mp4`, `.avi`, `.mov`, `.mkv` |

### 2.2 프로젝트 열기 / 목록 조회

| 항목 | 내용 |
|---|---|
| **설명** | 기존 프로젝트를 불러온다. |
| **표시 정보** | 프로젝트명, 생성일, 마지막 수정일, 진행률 (어노테이션 프레임 수 / 전체) |

### 2.3 프로젝트 삭제

| 항목 | 내용 |
|---|---|
| **설명** | 프로젝트와 관련 데이터를 삭제한다. |
| **동작** | 확인 다이얼로그 → SQLite DB 및 프로젝트 메타 삭제 |
| **조건** | 원본 비디오 파일은 삭제하지 않는다. |

## 3. 비디오 뷰어

### 3.1 비디오 재생 / 정지

| 항목 | 내용 |
|---|---|
| **설명** | 비디오를 재생하거나 정지한다. |
| **단축키** | `Space` |
| **속도 조절** | 0.25x, 0.5x, 1x, 2x, 4x |

### 3.2 프레임 탐색

| 항목 | 내용 |
|---|---|
| **설명** | 특정 프레임으로 이동한다. |
| **방법** | 타임라인 클릭, 프레임 번호 직접 입력, 키보드 방향키 |
| **단축키** | `←` / `→` : 이전/다음 프레임, `Shift+←/→` : ±10 프레임 |

### 3.3 타임라인

| 항목 | 내용 |
|---|---|
| **설명** | 전체 비디오 길이 대비 현재 위치 및 트랙 구간을 시각화한다. |
| **표시 정보** | 현재 프레임 커서, 트랙 구간 바, 키프레임 마커 |
| **인터랙션** | 드래그 스크러빙, 클릭 이동 |

### 3.4 프레임 정보 표시

| 항목 | 내용 |
|---|---|
| **설명** | 현재 프레임 번호, 타임스탬프(HH:MM:SS:ff), FPS, 해상도를 표시한다. |

## 4. 어노테이션 에디터

### 4.1 바운딩 박스 생성

| 항목 | 내용 |
|---|---|
| **설명** | 현재 프레임에 바운딩 박스 어노테이션을 추가한다. |
| **동작** | 캔버스 드래그 → 박스 생성 → 클래스 선택 → Detection 저장 |
| **단축키** | `B` : 박스 그리기 모드 |

### 4.2 바운딩 박스 편집

| 항목 | 내용 |
|---|---|
| **설명** | 기존 바운딩 박스의 위치 및 크기를 수정한다. |
| **동작** | 박스 선택 → 드래그 (이동) 또는 핸들 드래그 (크기 조절) |

### 4.3 바운딩 박스 삭제

| 항목 | 내용 |
|---|---|
| **설명** | 선택된 Detection을 삭제한다. |
| **단축키** | `Delete` / `Backspace` |

### 4.4 클래스 레이블

| 항목 | 내용 |
|---|---|
| **설명** | 각 Detection에 클래스 레이블을 할당한다. |
| **클래스 목록** | 프로젝트 생성 시 정의한 목록 (예: `person`, `vehicle`) |
| **표시** | 박스 상단에 클래스명 + Identity ID 오버레이 |

### 4.5 프레임 간 보간 (Interpolation)

| 항목 | 내용 |
|---|---|
| **설명** | 두 키프레임 사이의 Detection 위치를 선형 보간한다. |
| **동작** | 트랙 선택 → 시작/끝 프레임 키프레임 지정 → 자동 보간 |
| **결과** | 중간 프레임에 보간된 Detection이 자동 생성됨 |

### 4.6 키프레임 관리

| 항목 | 내용 |
|---|---|
| **설명** | Detection을 키프레임으로 지정하거나 해제한다. |
| **단축키** | `K` : 현재 프레임을 키프레임으로 지정 |
| **표시** | 타임라인의 해당 위치에 키프레임 마커 표시 |

## 5. Track / Identity 관리

### 5.1 Track 개념

Track은 하나의 연속 구간 내에서 단일 객체의 궤적이다.  
동일 객체가 화면에서 사라졌다가 재등장하면 별도의 Track이 생성된다.

### 5.2 Track 생성

| 항목 | 내용 |
|---|---|
| **자동** | AI 자동 레이블링 실행 시 생성됨 |
| **수동** | 첫 Detection 생성 시 자동으로 새 Track 할당 |

### 5.3 Track 목록 패널

| 항목 | 내용 |
|---|---|
| **표시 정보** | Track ID, 소속 Identity, 클래스, 시작/끝 프레임, Detection 수 |
| **인터랙션** | 클릭 선택 시 비디오 뷰어에서 해당 트랙 하이라이트 |

### 5.4 Identity 개념

Identity는 실세계의 단일 객체를 나타낸다.  
하나의 Identity는 여러 Track을 포함할 수 있다.

### 5.5 Identity 생성 / 삭제

| 항목 | 내용 |
|---|---|
| **생성** | 자동 (Track 생성 시 기본 1:1 매핑) 또는 수동 |
| **삭제** | Identity 삭제 시 소속 Track들은 Unassigned 상태로 변경 |

### 5.6 Track → Identity 할당 변경

| 항목 | 내용 |
|---|---|
| **설명** | 트랙이 잘못된 Identity에 연결된 경우 재할당한다. |
| **동작** | Track 선택 → Identity 선택 드롭다운 → 저장 |
| **용도** | Identity switch 교정 |

### 5.7 Track 병합

| 항목 | 내용 |
|---|---|
| **설명** | 같은 객체의 두 Track을 하나의 Track으로 병합한다. |
| **조건** | 두 Track의 프레임 구간이 겹치지 않아야 한다. |
| **동작** | Track A 선택 → Track B 선택 → 병합 실행 |

### 5.8 Track 분리

| 항목 | 내용 |
|---|---|
| **설명** | 하나의 Track을 특정 프레임을 기준으로 두 개로 분리한다. |
| **동작** | Track 선택 → 분리 프레임 지정 → 분리 실행 |
| **결과** | 원본 Track이 Track A (앞 구간), Track B (뒤 구간)로 나뉨 |

## 6. ReID Crop Gallery

### 6.1 크롭 이미지 추출

| 항목 | 내용 |
|---|---|
| **설명** | 각 Track의 Detection으로부터 크롭 이미지를 추출하여 표시한다. |
| **추출 기준** | 키프레임 또는 N 프레임마다 1장 (설정 가능) |
| **표시** | Identity별 그리드 갤러리 |

### 6.2 크롭 갤러리 탐색

| 항목 | 내용 |
|---|---|
| **설명** | Identity별로 분류된 크롭 이미지를 탐색한다. |
| **인터랙션** | 크롭 클릭 시 해당 프레임으로 비디오 이동 |
| **필터** | Identity ID, 클래스, 프레임 구간으로 필터 |

### 6.3 Identity 병합 (갤러리에서)

| 항목 | 내용 |
|---|---|
| **설명** | 크롭 갤러리에서 동일 인물로 판단되는 두 Identity를 병합한다. |
| **동작** | Identity A 선택 → Identity B 선택 → 병합 실행 |
| **결과** | Identity B의 모든 Track이 Identity A로 재할당됨 |

### 6.4 이상 크롭 감지 표시

| 항목 | 내용 |
|---|---|
| **설명** | 동일 Identity 내에서 시각적으로 이질적인 크롭을 강조 표시한다. |
| **방법** | ReID Embedding 거리 기반 (AI 워커가 계산) |
| **표시** | 갤러리에서 빨간 테두리 또는 경고 아이콘 |

## 7. AI 자동 레이블링

### 7.1 Detection 실행 (YOLO)

| 항목 | 내용 |
|---|---|
| **설명** | 선택한 프레임 범위에 대해 YOLO Detection을 실행한다. |
| **입력** | 모델 경로 (로컬 `.pt` 파일), 대상 클래스, 신뢰도 임계값, 프레임 범위 |
| **출력** | Detection 결과 → DB 저장 |
| **상태 표시** | 진행률 바, 처리 FPS, 예상 완료 시간 (SSE 스트리밍) |

### 7.2 Tracking 실행 (ByteTrack)

| 항목 | 내용 |
|---|---|
| **설명** | Detection 결과를 기반으로 ByteTrack 알고리즘을 적용하여 Track을 생성한다. |
| **입력** | Detection 결과, ByteTrack 파라미터 (track_thresh, match_thresh, track_buffer) |
| **출력** | Track 목록 및 Track-Detection 연결 → DB 저장 |
| **전제 조건** | 해당 프레임 범위의 Detection이 존재해야 한다. |

### 7.3 ReID Embedding 계산

| 항목 | 내용 |
|---|---|
| **설명** | 각 Detection 크롭에 대해 ReID Embedding을 계산하여 저장한다. |
| **입력** | ReID 모델 경로 (로컬 `.pth` 파일) |
| **출력** | 각 Detection에 Embedding 벡터 저장 |
| **용도** | 이상 크롭 감지, 향후 자동 Identity 매칭 |

### 7.4 워커 상태 모니터링

| 항목 | 내용 |
|---|---|
| **설명** | GPU 인퍼런스 워커의 현재 상태를 표시한다. |
| **상태 종류** | Idle, Running, Error |
| **표시 정보** | 현재 작업 이름, GPU 메모리 사용량, 진행률 |
| **통신** | Server-Sent Events (SSE) |

### 7.5 인퍼런스 취소

| 항목 | 내용 |
|---|---|
| **설명** | 실행 중인 AI 작업을 중단한다. |
| **동작** | 취소 버튼 클릭 → 워커 프로세스에 중단 신호 → 완료된 결과까지 저장 |

## 8. 데이터셋 내보내기

### 8.1 YOLO 포맷 내보내기

| 항목 | 내용 |
|---|---|
| **설명** | Detection 어노테이션을 YOLO 텍스트 포맷으로 내보낸다. |
| **출력 구조** | `images/`, `labels/`, `data.yaml` |
| **포맷** | `<class> <cx> <cy> <w> <h>` (정규화 좌표) |
| **옵션** | train/val 분할 비율, 포함할 클래스 선택 |

### 8.2 MOT Challenge 포맷 내보내기

| 항목 | 내용 |
|---|---|
| **설명** | 트래킹 어노테이션을 MOT Challenge 포맷으로 내보낸다. |
| **출력 구조** | `gt/gt.txt`, `seqinfo.ini` |
| **포맷** | `<frame>, <id>, <bb_left>, <bb_top>, <bb_width>, <bb_height>, <conf>, <x>, <y>, <z>` |
| **ID 매핑** | Identity ID를 MOT Track ID로 사용 |

### 8.3 내보내기 옵션

| 옵션 | 설명 |
|---|---|
| 프레임 범위 | 전체 또는 구간 지정 |
| 클래스 필터 | 특정 클래스만 포함 |
| 최소 Detection 수 | N개 미만인 Track 제외 |
| 신뢰도 임계값 | 해당 값 미만 Detection 제외 |

## 9. 비기능 요구사항

### 9.1 성능

| 항목 | 목표 |
|---|---|
| UI 응답 | 어노테이션 편집 조작 100ms 이하 |
| 비디오 탐색 | 프레임 이동 200ms 이하 |
| YOLO 인퍼런스 | GPU 환경에서 30 FPS 이상 (1080p 기준) |
| DB 쿼리 | 10,000 Detection 기준 50ms 이하 |

### 9.2 호환성

| 항목 | 사양 |
|---|---|
| OS | Linux (Ubuntu 20.04+), macOS (12+), Windows 10+ |
| 브라우저 | Chrome 100+, Firefox 100+, Edge 100+ |
| Python | 3.10 이상 |
| GPU | CUDA 11.8+ (선택, CPU fallback 지원) |

### 9.3 데이터 안전성

| 항목 | 내용 |
|---|---|
| 자동 저장 | 모든 어노테이션 변경은 즉시 DB에 저장 |
| 실행 취소 | 최소 20단계 Undo/Redo 지원 |
| 백업 | 내보내기 전 스냅샷 생성 옵션 |

### 9.4 확장성

| 항목 | 내용 |
|---|---|
| AI 모델 | 로컬 모델 파일 경로 지정으로 교체 가능 |
| 포맷 플러그인 | 내보내기 포맷 추가를 위한 플러그인 구조 (v0.2 목표) |
| 멀티 카메라 | DB 스키마 설계 시 카메라 ID 컬럼 포함 (향후 확장 고려) |
